## Refined Acceptance Criteria (All 4 ACs remain valid)

### AC 1: Elena assembles a Test from three ATCs

Given Elena is signed in to workspace "Acme QA" with role "member"
When Elena assembles a Test with title "Add to Cart from Empty State" from three ATCs [open-cart, click-add, verify-count]
Then the Test is created, appears in Test list, preserves ATC order, and activity log records Test.create event

### AC 2: Saving a Test without any ATC is blocked

Given Elena is on the "New Test" form with title but no ATCs
When she clicks "Save"
Then save is blocked (400/422), error message shows "A Test must include at least one ATC", no Test created, no activity log entry

### AC 3: Double-click on Save button within idempotency window

Given Elena fills the form with valid data and clicks "Save"
When she clicks "Save" again within 1 second (before response arrives)
Then exactly one Test is created, activity log records one event, second request returns same Test ID

### AC 4: Elena cannot use ATCs from a workspace she does not belong to

Given Elena belongs to "Acme QA" but NOT to "Other Co"
When Elena attempts to create a Test in "Acme QA" with a foreign ATC from "Other Co"
Then creation is rejected (403/400), error says "You do not have permission to use this ATC" (no leakage), no Test created
